			Java – Programação Orientada a Objetos (POO)

	Conceitos básicos da linguagem
Java é case sensitive, CamelCase e Orientação a Objetos pura.

CamelCase: Classes e interfaces têm letra maiúscula em cada palavra. Métodos e atributos começam com minúscula e depois maiúscula, por exemplo calcularMedia.

Main mais limpo possível, focada na lógica e interação com o usuário. Usar classes de gerenciamento e criação de objetos.

Single Responsiblity Principle: um dos cinco princípios do SOLID. Uma classe deve ter apenas uma razão para mudar, significando que uma classe deve ter apenas um trabalho ou responsabilidade.


	Packages
Package define o pacote, que é uma forma de organizar classes.
Funciona como uma pasta lógica dentro do projeto.
Permite reutilizar código usando import.
Exemplo: package poojava;


	Tipos de dados
Tipos primitivos guardam valores diretamente na memória e são mais rápidos. Principais tipos primitivos:
int para números inteiros                     | double e float para números reais.
boolean para verdadeiro ou falso              | char para um único caractere.
long e short para inteiros maiores ou menores | void indica que um método não retorna valor.
final 					      | Modificador constante, como o const.

Tipos por referência guardam o endereço de memória (heap).
Exemplos: String, arrays e objetos.

String não é tipo primitivo, é uma classe.
Possui métodos como length, isEmpty, toUpperCase, toLowerCase.


	Bibliotecas (imports)
| import java.X.Y; | X: util para utilidades, io para in/out, time para data-hora, etc.

InputMismatchException / FileNotFoundException / IOException | Erros.
LocalDate / LocalTime / LocalDateTime / DateTimeFormatter    | Data e hora.
Arrays / ArrayList / HashMap / LinkedList / List / Vector    | Listas.
FileWriter / BufferedReader / FileReader 		     | Leitura e escrita em arquivo.
Scanner 						     | Leitura do usuário.
Random  						     | Aleatoriedade.


	Manipulação de String
Os principais métodos são:

length retorna o tamanho do texto.
charAt retorna o caractere em uma posição.
indexOf e lastIndexOf encontram posições.
replace substitui texto.
toUpperCase e toLowerCase mudam para maiúsculo ou minúsculo.
contains verifica se contém texto.
isEmpty verifica se está vazio.


	Estruturas de decisão
if e else executam blocos dependendo de uma condição.

Operador ternário é uma forma curta de if e else.
Exemplo: condição ? valorSeVerdadeiro : valorSeFalso

switch é usado quando há várias opções possíveis.
default executa quando nenhum caso é atendido.


	Estruturas de repetição
while repete enquanto a condição for verdadeira.
do while executa pelo menos uma vez antes de testar a condição.
for é usado quando se sabe quantas vezes repetir.

break interrompe o loop.
continue pula para a próxima repetição.


	Métodos
Métodos servem para reutilizar código. Podem retornar valor ou não.
Sobrecarga de método acontece quando existem métodos com o mesmo nome, mas parâmetros diferentes.
Varargs permitem passar quantidade variável de argumentos para um método.


	Tratamento de erros (exceções)
try contém o código que pode gerar erro.
catch trata o erro.
finally sempre executa, com erro ou não.
Evita que o programa quebre.


	Arquivos de texto
FileWriter escreve arquivos pequenos.
BufferedWriter é melhor para arquivos grandes.
PrintWriter é usado para relatórios e logs.
FileReader e BufferedReader leem arquivos.
FileOutputStream é usado para arquivos binários como imagens e áudios.


	Programação Orientada a Objetos
Classe é o modelo. Objeto é uma instância da classe.

Encapsulamento esconde partes da classe e controlar quem pode acessar.
public    | Pode ser acessado de qualquer lugar do programa.
private   | Só pode ser acessado dentro da própria classe. getters e setters.
protected | Pode ser acessado na mesma classe, no mesmo pacote e em classes filhas(herança).
static	  | Pertence á classe, não ao objeto. Será algo em comum em todas intâncias.


	Herança
Uma classe filha herda características da classe pai.
Usa a palavra extends.

'super' é usado para chamar o construtor ou métodos da classe pai.

	Polimorfismo
Um objeto pode ser tratado como se fosse de outro tipo.
Muito usado com herança e classes pai.

	Classes abstratas
Não podem ser instanciadas.
Servem como modelo para classes filhas.
Classes filhas são obrigadas a implementar métodos abstratos.

	Interface
Parecida com classe abstrata.
Uma classe pode implementar várias interfaces.
Usa a palavra implements.

	Static
Pertence à classe e não ao objeto.
É compartilhado entre todas as instâncias.

	toString
Converte um objeto em texto.
Quando se usa System.out.println(obj), o Java chama toString automaticamente.
Pode ser sobrescrito para mostrar informações do objeto.


	Wrapper Classes
Permitem tratar tipos primitivos como objetos.
Exemplos:
int vira Integer
double vira Double
boolean vira Boolean

Autoboxing converte automaticamente primitivo para objeto.
Unboxing faz o contrário.


	Enum
Representa um conjunto fixo de constantes.
Mais seguro que usar String para opções fixas.


	Threads
Executam tarefas simultaneamente.
Ajudam no desempenho.
Multithreading significa vários fluxos de execução ao mesmo tempo.


	Agregação e Composição
Agregação ocorre quando objetos podem existir de forma independente.
Composição ocorre quando um objeto depende do outro.
Exemplo de composição: Moto possui Engrenagem.


		Collection Framework e Map
Conjunto unificado de classes e interfaces que permite manipular dados. Fornece Estruturas de Dados prontas.
For-each percorre todos os elementos de um array.
Collection > Set   > HashSet
           > List  > Vector > Stack
		   > ArrayList
		   > LinkedList
           > Queue >  ^
Map 	   > HashMap

	SET
HashSet: Não permite repetições. Melhor opção para busca rápida, não mantendo os elementos em nenhuma ordem específica.

	Map
HashMap: Mapeia chaves únicas a valores. Permite valores duplicados. Busca rápida com duplicidade de valores.
Processa o valor de entrada(chave) com a função hashCode() e o valor resultante é "bucket" em que a chave será armazenada. Se 2+ chaves tiverem mesmo hash code, temos uma colisão. HashMap armazenará esses pares no mesmo bucket como uma lista encadeada ou uma árvore balanceada quando muitas colisões em um só bucket.

	Queue
LinkedList: Funciona como fila. O primeiro que entra é o primeiro que sai (FIFO – First In, First Out).
Stack: Pilha de pratos. O primeiro aentrar é o último a sair(FILO – First In, Last Out).

	List
Coleção que permite elementos duplicados. Permite acesso posicional e inserção de elementos em pontos arbitrários da lista.

Array: é uma estrutura de tamanho fixo. Array 2D é um array que guarda outros arrays, formando uma matriz.
ArrayList (lista contigua): array dinâmico, cresce conforme necessário, acesso rápido via índice.
LinkedList (lista encadeada): melhor performance em inserção e remoção em qualquer ponto da lista.
Vector (lista sincronizada): parecido com ArrayList, mas seguro para múltiplas threads e um pouco mais lento.


		Princípio da Responsabilidade Única (Single Responsiblity Principle - SRP)
É um dos cinco princípios fundamentais do SOLID. Declara que uma classe deve ter apenas uma razão para mudar, significando que uma classe deve ter apenas um trabalho ou responsabilidade.

Gerenciador de classes: é uma classe cuja responsabilidade é criar, armazenar, buscar, atualizar e remover objetos de outra classe, pois sem ela, o Main - que deveria apenas chamar classes - ficaria responsável por tudo isso e violaria o SRP. Usa quando tem múltiplas instâncias de uma classe e precisa consultar, organizar ou manipular dados.


		Relacionamentos: Associação
É a relação do tipo "tem um", a classe Carro tem uma classe Roda. Carro tem o atributo List<Roda> rodas.
Tendo uma classe que gerencia outra, teremos os metodos Buscar, Remover, Adicionar, toString com StringBuilder e copiar adaptados a uma lista de objetos gerenciada.

		Associação + Gerenciador + SRP
